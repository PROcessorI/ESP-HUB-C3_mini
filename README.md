# ESP-HUB — Универсальный сенсор-хаб с управлением светильниками для ESP32

Лёгкая многопротокольная платформа телеметрии для ESP32: подключайте датчики по GPIO, I2C, 1-Wire, UART или CAN, настраивайте через веб-интерфейс, отправляйте данные по MQTT, HTTP, CAN или UART, управляйте светильниками с расписанием.

## Возможности

### Датчики и телеметрия
- **8 слотов для датчиков** — DHT11/22, BMP280, BH1750, DS18B20, Analog, MH-Z19B, SDS011, UART Generic, CAN RAW
- **Мультишинная архитектура** — каждый слот независимо работает на шине Auto / GPIO / I2C / 1-Wire / UART / CAN
- **Мультипротокольный вывод** — каждый слот независимо отправляет данные через MQTT / HTTP POST / CAN TX / Serial

### Управление светильниками
- **4 канала RGB** — красный, дальний красный, синий, белый (шкала 0-200, каждый шаг 0.5%)
- **8 расписаний (сценариев)** — включение светильников в точное время суток (по NTP или Backup Timer)
- **8 интервальных таймеров** — повторяющиеся действия с настраиваемым интервалом (**работают всегда, без интернета!**)
- **Режимы** — OFF, GROW, FULL, RED, BLUE, PULSE и свои настройки
- **Резервный таймер (Backup Timer)** — для сценариев, когда нет интернета

### Веб-интерфейс и API
- **Веб-интерфейс** — Dashboard, настройка датчиков, MQTT/WiFi, информация о системе
- **REST API** — управление датчиками, светильниками и GPIO через HTTP
- **MQTT-телеметрия** — автоматическая отправка JSON с настраиваемым интервалом
- **Тёмная / светлая тема** — переключается в шапке, запоминается в браузере
- **Русский / английский язык** — переключается в шапке без перезагрузки

### � ESP-CAM интеграция
- **Просмотр потока** — MJPEG стрим с ESP32-CAM прямо в браузере (страница `/camera`)
- **Управление камерой** — разрешение (96×96 … UXGA), качество JPEG, яркость, зеркалирование по горизонтали/вертикали
- **Вспышка** — переключение вспышки одной кнопкой
- **Снимок** — сохранение JPEG кадра напрямую с камеры
- **Выбор устройства записи** — Не записывать / SD Card на ESP32-CAM / MQTT (JPEG base64)
- **Relay-API** — ESP-HUB проксирует команды к ESP32-CAM по `/api/camera/relay`, обходя ограничения CORS браузера

### Система и сетевые возможности
- **WiFi + AP fallback** — при неудачном подключении к сети запускает свою точку доступа
- **Captive portal** — при подключении к AP браузер автоматически открывает интерфейс (работает с интернетом и без)
- **WiFi-сканер** — поиск доступных сетей прямо из веб-интерфейса
- **LittleFS** — настройки хранятся в JSON и переживают перезагрузку
- **AJAX обновления** — живые данные на Dashboard без перезагрузки страницы
- **GPIO таймеры** — автоматические переключения выводов по расписанию
- **BLE** — Bluetooth Low Energy для мобильных приложений

## Поддерживаемые датчики

| Датчик           | Шина    | Показания                          | Поле `pin`                           |
|------------------|---------|------------------------------------|--------------------------------------|
| DHT11            | GPIO    | Температура, Влажность             | GPIO пин данных                      |
| DHT22            | GPIO    | Температура, Влажность             | GPIO пин данных                      |
| BMP280           | I2C     | Температура, Давление, Высота      | I2C-адрес (дес.), напр. 118 = 0x76   |
| BH1750           | I2C     | Освещённость (lux)                 | I2C-адрес (дес.), напр. 35 = 0x23    |
| DS18B20          | 1-Wire  | Температура                        | GPIO пин OneWire                     |
| Analog           | GPIO    | Raw-значение, Напряжение           | GPIO пин АЦП                         |
| **MH-Z19B**      | **UART**| CO₂ (ppm), Температура             | GPIO RX; TX задаётся в `pin2`        |
| **SDS011**       | **UART**| PM2.5 (μg/m³), PM10 (μg/m³)       | GPIO RX; TX задаётся в `pin2`        |
| **UART Generic** | **UART**| До 4 значений (JSON или CSV)       | GPIO RX; TX задаётся в `pin2`        |
| **CAN RAW**      | **CAN** | До 4 int16-пар из фрейма           | CAN ID задаётся отдельно             |

> I2C-датчики используют пины SDA=21, SCL=22 (стандартные для ESP32).  
> UART-датчики используют HardwareSerial 1 или 2 (выбирается в настройках слота).  
> CAN-шина использует встроенный TWAI-контроллер ESP32; пины TX/RX по умолчанию GPIO5/GPIO4.

## Быстрый старт

### 1. Требования

- [PlatformIO](https://platformio.org/) (CLI или расширение для VS Code)
- Плата ESP32 (esp32dev или совместимая)
- USB-кабель для прошивки

### 2. Клонирование и сборка

```bash
git clone <repo-url> ESP-HUB
cd ESP-HUB
pio run
```

### 3. Прошивка

```bash
pio run --target upload
```

### 4. Первый запуск

1. После прошивки ESP32 не найдёт сохранённую WiFi-сеть и создаст точку доступа:
   - **SSID**: `ESP-HUB`
   - **Пароль**: `12345678`

2. Подключитесь к этой сети и откройте **`http://192.168.4.1`** в браузере

3. На странице **MQTT** введите:
   - SSID и пароль вашей WiFi-сети (кнопка Scan)
   - Хост MQTT-брокера, порт (обычно 1883)
   - Имя пользователя и пароль MQTT (если требуется)

4. На странице **Sensors** включите нужные датчики:
   - Выберите тип датчика из выпадающего списка
   - Укажите GPIO пин или I2C-адрес
   - Задайте метку (user-friendly имя)
   - Выберите протокол вывода (MQTT, HTTP, CAN, Serial)

5. На странице **Light** (если используется контроллер светильника):
   - Можно вручную управлять яркостью каналов
   - Настроить расписания (сценарии)
   - Настроить интервальные таймеры
   - Можно скрывать/показывать строки сценариев и таймеров, но остаётся хотя бы одна (последнюю нельзя убрать)

6. Нажмите **Save** → ESP32 перезагрузится и подключится к вашей сети

### 5. Управление через Serial (USB)

Подключитесь к COM-порту на скорости **115200 бод**:

```bash
# Минимум управления
> help              # Показать справку по командам
> status            # Статус системы
> sensors           # Показать показания датчиков
> clock             # Показать статус времени (NTP / Backup Timer)
> light on          # Включить светильник
> light grow        # Включить режим роста
> scenario list     # Показать расписания
```

**Примеры команд:**
```bash
> R100 B50          # Красный 100%, синий 50%
> scenario enable 0 # Включить сценарий #0
> wifi scan         # Сканировать сети
> auto 5            # Вывод датчиков каждые 5 сек
> clock             # Проверить время (NTP или Backup Timer)
```

➡️ Полный справочник команд: [**COMMANDS.md**](./COMMANDS.md)

### Шины и протоколы вывода

Каждому слоту датчика назначаются независимо:

| Параметр | Варианты | Описание |
|---|---|---|
| **Bus (шина)** | Auto, GPIO, I2C, 1-Wire, UART, CAN | Физический интерфейс датчика. Auto = определяется по типу датчика. |
| **Output (вывод)** | MQTT, HTTP POST, CAN TX, Serial | Куда отправляются данные при каждом считывании. |

**HTTP POST** — отправляет на указанный URL JSON:
```json
{"device":"ESP-HUB","slot":3,"label":"dust","data":{"pm2.5":12.3,"pm10":18.7}}
```

**CAN TX** — кодирует float-значения как int16 (×10) и отправляет CAN-фрейм с заданным ID.

**Serial** — выводит в Serial Monitor (115200 бод) строку формата:
```
[SENSOR] slot=3 label=dust pm2.5=12.30 pm10=18.70
```

## Резервный таймер (Backup Timer) — Работа без интернета

### Что это такое?

**Backup Timer** обеспечивает работу расписаний светильников **даже при отсутствии интернет-соединения**. Используется двухуровневая система времени:

| Режим | Источник | Точность | Требует интернет | Примечания |
|-------|----------|----------|---------|----------|
| **NTP** | Сервер времени | ±1 сек | ✅ Да | Используется когда WiFi подключена |
| **Backup Timer** | ESP32 millis() | ±0.5% | ❌ Нет | Используется когда нет интернета |

### Как это работает

1. **При наличии интернета**
   - WiFi подключена → синхронизация с NTP сервером
   - Расписания используют точное время
   - ✅ Все работает идеально

2. **При потере интернета**
   - Система автоматически переходит на Backup Timer
   - Счет времени ведется от момента включения устройства
   - ✅ Расписания продолжают работать
   - ⚠️ Время при перезагрузке сбросится (если нет RTC модуля)

3. **При восстановлении интернета**
   - Автоматическая ресинхронизация с NTP
   - Плавный переход без сбоев

### Команды диагностики

```bash
# Проверить статус часов
> clock

# Вывод с интернетом (NTP):
# Current Time     : 14:30:45 (2025-03-11)
# NTP Status       : SYNCED

# Current Time     : 14:30:45 (1970-01-12)

# Backup Time      : 0d 14:30:45 (uptime based)
```

## Как добавить новый датчик — пошаговая инструкция

Допустим, вы хотите добавить датчик **SHT31** (I2C, температура + влажность).

### Шаг 1. Добавить библиотеку в `platformio.ini`

```ini
lib_deps =
    ...
    adafruit/Adafruit SHT31 Library@^2.2.2    ; ← добавить
```

### Шаг 2. Добавить тип датчика в перечисление (`src/config.h`)

Откройте `src/config.h` и добавьте новый элемент **перед** `SENSOR_TYPE_COUNT`:

```cpp
enum SensorType : uint8_t {
    SENSOR_NONE = 0,
    SENSOR_DHT11, SENSOR_DHT22,
    SENSOR_BMP280, SENSOR_BH1750, SENSOR_DS18B20, SENSOR_ANALOG,
    SENSOR_MHZ19, SENSOR_SDS011, SENSOR_UART_GENERIC, SENSOR_CAN_RAW,
    SENSOR_SHT31,          // ← добавить
    SENSOR_TYPE_COUNT       // всегда последний!
};
```

### Шаг 3. Добавить имя в массив имён (`src/config.cpp`)

```cpp
static const char* _sensorNames[] = {
    "None","DHT11","DHT22","BMP280","BH1750","DS18B20","Analog",
    "MHZ19","SDS011","UART Generic","CAN RAW",
    "SHT31"   // ← добавить
};
```

> **Важно:** порядок строк должен совпадать с порядком в `SensorType`.

### Шаг 4. Создать драйвер датчика

Создайте файл `src/sensors/sht31_sensor.h`. Драйвер наследуется от `SensorBase`:

```cpp
#pragma once
// ============== SHT31 Temperature + Humidity ==============

#include "sensor_base.h"
#include <Wire.h>
#include <Adafruit_SHT31.h>

class SHT31Sensor : public SensorBase {
public:
    // addr: I2C-адрес (0x44 по умолчанию, 0x45 — альтернативный)
    SHT31Sensor(uint8_t addr = 0x44) : _addr(addr) {}

    bool begin() override {
        _ready = _sht.begin(_addr);
        if (_ready) Serial.printf("[SHT31] OK at 0x%02X\n", _addr);
        else        Serial.printf("[SHT31] FAIL at 0x%02X\n", _addr);
        return _ready;
    }

    uint8_t read() override {
        clearValues();
        if (!_ready) return 0;

        float t = _sht.readTemperature();
        float h = _sht.readHumidity();

        uint8_t count = 0;
        if (!isnan(t)) { setVal(0, "temperature", "\xC2\xB0""C", t); count++; }
        if (!isnan(h)) { setVal(1, "humidity",    "%",            h); count++; }
        return count;
    }

    uint8_t valueCount() const override { return 2; }
    const SensorValue& getValue(uint8_t idx) const override { return _values[idx]; }
    const char* typeName() const override { return "SHT31"; }

private:
    uint8_t _addr;
    Adafruit_SHT31 _sht;
};
```

#### Обязательные виртуальные методы `SensorBase`

| Метод | Что должен делать |
|---|---|
| `bool begin()` | Инициализация оборудования. Установить `_ready = true/false`. Вернуть `true` при успехе. |
| `uint8_t read()` | Считать данные. Вызвать `clearValues()` в начале, затем `setVal(idx, name, unit, value)` для каждого значения. Вернуть количество валидных значений. |
| `uint8_t valueCount() const` | Сколько значений возвращает датчик (например, DHT = 2, BMP280 = 3). |
| `const SensorValue& getValue(uint8_t idx) const` | Вернуть `_values[idx]`. |
| `const char* typeName() const` | Имя для отладки и логов. |

#### Вспомогательные методы базового класса

- `clearValues()` — обнуляет все `_values`, ставит `valid = false`
- `setVal(idx, name, unit, value)` — заполняет `_values[idx]`: имя, единицу измерения, значение, ставит `valid = true`

#### Структура `SensorValue`

```cpp
struct SensorValue {
    char name[16];   // "temperature", "humidity", "pressure" и т.д.
    char unit[8];    // "°C", "%", "hPa", "lx"
    float value;     // числовое значение
    bool valid;      // true если измерение успешно
};
```

> Максимум `MAX_VALUES_PER_SENSOR = 4` значения на один датчик.

### Шаг 5. Зарегистрировать драйвер в фабрике (`src/sensor_manager.cpp`)

Добавьте `#include` и новый `case` в `createSensor()`:

```cpp
#include "sensors/sht31_sensor.h"     // ← добавить наверху

// внутри switch (sc.type):
case SENSOR_SHT31: return new SHT31Sensor(sc.pin);  // ← добавить
```

### Шаг 6. Собрать и прошить

```bash
pio run --target upload
```

Открыть веб-интерфейс → Sensors → выбрать SHT31 в одном из слотов → указать I2C-адрес (0x44) → Save.

**Готово!** Новый датчик появится на Dashboard и в MQTT-телеметрии.

## Что НЕ нужно менять при добавлении датчика

| Компонент | Почему не нужно менять |
|---|---|
| **Веб-интерфейс** | Список типов датчиков генерируется из `SENSOR_TYPE_COUNT` и `sensorTypeName()`. Новый тип появится в выпадающем списке автоматически. |
| **Dashboard** | AJAX-запрос `/api/data` вызывает `buildJson()`, который опрашивает все `_sensors[]` через абстрактный интерфейс `SensorBase`. |
| **MQTT** | `SensorManager::buildJson()` формирует JSON из `SensorValue`, используя `getValue()`. Новые поля появятся автоматически. |
| **Конфигурация** | `ConfigManager` сохраняет/загружает `SensorType` как число. Новый элемент enum сериализуется без изменений. |

## Веб-интерфейс

| Страница | URL | Описание |
|---|---|---|
| **Dashboard** | `/` | Живые данные датчиков (обновление раз в 2 сек), статус WiFi/MQTT |

| **System** | `/system` | Информация о чипе, аптайм, свободная память, перезагрузка, сброс настроек |

## Настройка AP (точки доступа)

По умолчанию:
- SSID: `ESP-HUB`
- Пароль: `12345678`

AP запускается автоматически, если ESP32 не может подключиться к WiFi за 15 секунд. В режиме AP доступен веб-интерфейс на `192.168.4.1`.

Чтобы изменить SSID/пароль AP, отредактируйте значения по умолчанию в `HubConfig` (`src/config.h`).

## REST API

ESP-HUB предоставляет полный **REST API** для управления датчиками, светильником, GPIO и системой. Используйте API для интеграции с домашней автоматизацией, мобильными приложениями или внешними системами.

### Основные эндпоинты

| Endpoint | Метод | Описание |
|---|---|---|
| `/api/data` | GET | Получить текущие показания датчиков (JSON) |
| `/api/system` | GET | Статус ESP32 (heap, uptime, flash, CPU) |
| `/api/wifi` | GET | Статус подключения WiFi |
| `/api/mqtt` | GET | Статус MQTT-брокера |
| `/api/scan` | GET | Сканировать доступные WiFi-сети |
| `/api/fixture` | GET | Статус светильника (RGBW) |
| `/api/fixture/set` | POST | Установить каналы светильника |
| `/api/fixture/on` | POST | Включить светильник |
| `/api/fixture/off` | POST | Выключить светильник |
| `/api/fixture/toggle` | POST | Toggle светильника |
| `/api/fixture/dim` | POST | Затемнить/осветлить |
| `/api/fixture/scenarios` | GET | Получить расписание сценариев |
| `/api/fixture/timers` | GET | Получить интервальные таймеры |
| `/api/gpio` | GET | Статус GPIO-автоматики |
| `/api/gpio/set` | POST | Установить GPIO пин (HIGH/LOW) |

### Примеры

**cURL — установить цвет светильника**:
```bash
curl -X POST http://192.168.1.100/api/fixture/set \
  -d "red=100&blue=150&white=0&far_red=50"
```

**JavaScript — получить показания датчиков**:
```javascript
fetch('http://192.168.1.100/api/data')
  .then(r => r.json())
  .then(data => console.log(data));
```

**Python — включить GPIO импульс**:
```python
import requests
requests.post('http://192.168.1.100/api/gpio/set', data={'pin': 12, 'value': 1})
```

## FAQ

**Q: Как сбросить настройки?**
A: Откройте страницу System в веб-интерфейсе и нажмите «Factory Reset», или удалите файл `/config.json` из LittleFS.

**Q: Можно использовать другую плату ESP32?**
A: Да, измените `board = esp32dev` в `platformio.ini` на вашу плату (например, `esp32-s3-devkitc-1`).

**Q: Как добавить второй I2C-датчик одного типа?**
A: Используйте два разных слота с одинаковым типом, но разными адресами (например, BH1750: 0x23 и 0x5C).

**Q: MQTT не подключается — что делать?**
A: Проверьте через Serial Monitor (115200 бод). Логи покажут ошибки подключения. Убедитесь, что брокер доступен из сети ESP32.

**Q: Что будет, если отключить интернет при включённом расписании светильника?**
A: Расписание продолжит работать благодаря **Backup Timer**! Время отсчитывается по встроенным часам устройства (±0.5% точность). При перезагрузке без интернета время сбросится на `1970-01-01`.

**Q: Как сохранить точное время при перезагрузке без интернета?**
A: Подключите RTC модуль (DS3231). Интеграция уже подготовлена в коде (`src/system_clock.cpp`), нужно лишь раскомментировать методы `_loadFromRTC()` и `_saveToRTC()`.

**Q: Как проверить, работает ли Backup Timer?**
A: Используйте команду Serial консоли `clock`. Она покажет, используются ли NTP часы или Backup Timer.

**Q: Чем отличаются сценарии от интервальных таймеров?**
A: Сценарии срабатывают в конкретное время суток (например, в 08:00 и 20:00), используют системные часы (NTP или Backup Timer). Интервальные таймеры срабатывают через заданный интервал (например, каждые 15 минут), работают на millis() и не зависят от интернета.

**Q: Работают ли интервальные таймеры без интернета?**
A: Да! Интервальные таймеры используют внутренний счётчик `millis()` ESP32 и работают всегда, даже без интернета и без NTP синхронизации. Это их главное отличие от сценариев.

**Q: Как настроить интервальный таймер с автовыключением?**
A: В веб-интерфейсе на странице Light: задайте интервал (h:m:s), выберите действие (CUSTOM для своих значений), укажите каналы (R/FR/B/W), и задайте "Run Duration" — время работы до автоматического выключения.
