# Внутренний Резервный Таймер (Backup Timer) и Интервальные Таймеры

## Описание

**Backup Timer** — это внутренний регулятор времени, обеспечивающий работу расписаний (сценариев) светильников и других функций, требующих точного времени, **даже при отсутствии интернет-соединения**.

ESP-HUB использует **две независимые системы таймеров**:

### 1. Сценарии по времени (Time-based Scenarios)
- Срабатывают в **конкретное время суток** (например, 08:00:00, 20:00:00)
- Используют **системные часы** (NTP или Backup Timer)
- 8 слотов для сценариев

### 2. Интервальные таймеры (Interval Timers) 
- Срабатывают через **заданный интервал** (например, каждые 15 минут)
- Используют **внутренний счётчик `millis()`** — работают всегда, независимо от интернета!
- 8 слотов для таймеров
- Поддерживают импульсные режимы и автовыключение

---

## Система времени для сценариев

Система работает в двух режимах:

1. **Режим NTP (Normal)** — если подключение к интернету установлено
   - Синхронизируется с серверами NTP (Network Time Protocol)
   - Точность: ±1 секунда
   - Используется для всех расписаний и сценариев

2. **Режим Backup** — если интернета нет или NTP не подключилась
   - Использует внутренний счетчик системы (`millis()`)
   - Время отсчитывается с момента включения системы
   - **Важно**: при перезагрузке ESP32 время сбросится!
   - Асинхронный переход: при восстановлении интернета автоматически перейдет на NTP

## Как это работает

### 1. Инициализация при старте

```cpp
// В main.cpp при загрузке:
systemClock.begin();  // Инициализация системных часов
```

Система проверяет:
- Доступность RTC (реального времени, если модуль подключен)
- Статус NTP при подключении WiFi

### 2. Синхронизация с NTP

Когда WiFi подключена и имеется интернет:

```cpp
// В wifi_manager.cpp:
time_t ntpTime;
time(&ntpTime);
systemClock.syncFromNTP(ntpTime);  // Синхронизировать часы
```

На консоли видно:
```
[SYSCLOCK] Synced with NTP: 1000000000 (seconds since epoch)
[SYSCLOCK] Current time: 2025-03-11 14:30:45
```

### 3. Backup режим при отсутствии интернета

Если NTP недоступна, система использует `millis()` для отсчета времени:

```cpp
// Интерно работает так:
time_t now() {
    if (ntpSynced) {
        return lastNTPTime + (millis() - lastNTPMillis) / 1000;
    } else {
        return bootTime + (millis() - backupStartMillis) / 1000;  // BACKUP
    }
}
```

Вывод в консоли:
```
[SYSCLOCK] BACKUP TIMER: 0d 14:30:45 | waiting for NTP...
```

### 4. Сценарии работают в обоих режимах

**Fixture Scenarios** (расписание светильников) работают независимо от наличия интернета:

```cpp
// В fixture_manager.cpp:
systemClock.getLocalTime(&timeinfo);  // Получить текущее время
// Проверить: совпадает ли текущее время (HH:MM:SS) с расписанием?
if (timeinfo.tm_hour == scenario.start_hour &&
    timeinfo.tm_min == scenario.start_minute &&
    timeinfo.tm_sec == scenario.start_second) {
    // ← Активировать сценарий! Работает и с NTP, и с Backup
    activateScenario();
}
```

## Как сценарии работают без интернета

### Пример конфигурации

Вы настроили в веб-интерфейсе:

| Сценарий | Время вкл. | Каналы | Статус |
|----------|-----------|--------|--------|
| 0        | 08:00:00  | R100 B100 | Enabled |
| 1        | 20:00:00  | R200 | Enabled |

### Сценарий: потеря интернета

1. **14:30:00** — нет интернета (WiFi отключена, нет NTP)
   ```
   [FIXTURE] Using BACKUP timer for scenarios (no NTP).
   [SYSCLOCK] BACKUP TIMER: 0d 14:30:00 | waiting for NTP...
   ```

2. **20:00:00** (Backup time) — система проверяет расписание по внутренним часам
   ```
   [FIXTURE] >>> SCENARIO 1 FIRED at 20:00:00 -> R=200 FR=0 B=0 W=0
   ```
   ✅ Сценарий сработал, несмотря на отсутствие интернета!

3. **23:45:00** — интернет восстановлен
   ```
   [NTP] Time synced: 23:45:30 2025-03-11
   [SYSCLOCK] Synced with NTP: 1000000000 (seconds since epoch)
   ```

4. **23:45:30** — система переходит на точное NTP время
   - Все последующие сценарии работают с микро-точностью
   - Если Backup Timer показывал 23:45:30, NTP показал 23:45:32 — нет проблемы!

## Нюансы и ограничения

### ⚠️ Взаимодействие с промежуточной синхронизацией

Если несколько раз теряется и восстанавливается интернет:

```
14:30  → NTP synced
15:00  → WiFi отключена (Backup timer тикает)
15:30  → WiFi восстановлена (NTP ресинхронизирована)
```

✅ Нет проблем — система отслеживает расхождения автоматически.

### ⚠️ Перезагрузка в режиме Backup

Если вы перезагрузили ESP32 **без интернета**:

```
[BOOT] Backup timer active: YES (waiting for NTP)
[SYSCLOCK] Current time: 1970-01-01 00:00:00   ← Time reset!
```

- Внутренний счетчик сбросился
- Сценарии не сработают, пока не восстановится NTP
- **Решение**: подключить интернет, система синхронизируется автоматически

### ⚠️ Дрейф времени в Backup режиме

Системный счетчик `millis()` имеет погрешность ~±0.5% на некачественных плат:

- За 1 час: ±18 сек дрейфа (редко)
- За 1 день: ±7 минут дрейфа (редко)

**Практически**: для большинства светильников это не критично. Если нужна идеальная точность — как можно раньше восстановите интернет.

## Расширения в будущем

### RTC (Real-Time Clock) модуль

Можно добавить DS3231 или другой RTC на I2C:

```cpp
// Методы уже заготовлены в SystemClock:
void _loadFromRTC();   // Загрузить время из модуля
void _saveToRTC(time_t t);  // Сохранить время в модуль
```

После интеграции:
- Время сохранится при перезагрузке **даже без интернета**
- При следующем включении Backup Timer будет точен
- RTC обычно потребляет <0.5 мА от батарейки CR2032

### NTP сервер на ESP32

Можно сделать устройство сервером времени для других гаджетов в локальной сети (если ESP32 знает точное время от основного маршрутизатора).

## Вопросы и ответы

**Q: Почему время в Backup режиме пилит вверх/вниз?**  
A: Это нормально — система просто считает время от включения. Если нужна точность — восстановите интернет.

**Q: Что будет, если я перезагружу ESP32 без интернета?**  
A: Время сбросится на `1970-01-01 00:00:00`. Сценарии не сработают, пока не восстановится интернет.

**Q: Зачем нужны сценарии, если есть интервальные таймеры?**  
A: Таймеры работают по интервалу (нужно для периодических действий).  
Сценарии срабатывают в конкретное время суток (для расписания по часам).

**Q: Может ли Backup Timer накапливать ошибку?**  
A: Редко. Максимум ~7 минут за день на дешевых платах. На хороших платах — <1 минуты в день.

# Интервальные Таймеры (Interval Timers)

## Что это такое?

**Интервальные таймеры** — это периодические действия, которые срабатывают через заданный интервал времени (часы:минуты:секунды). В отличие от сценариев, которые работают в конкретное время суток, интервальные таймеры работают на основе внутреннего счётчика `millis()` ESP32.

**Ключевое отличие**: Интервальные таймеры **работают всегда**, даже без интернета и без NTP синхронизации! Они используют внутренний счётчик времени, который тикает с момента включения ESP32.

## Как это работает

```cpp
// В fixture_manager.cpp:
unsigned long now_ms = millis();
for (int i = 0; i < MAX_FIXTURE_TIMERS; i++) {
    if (!_cfg->timers[i].enabled) continue;
    
    // Интервал в миллисекундах
    uint32_t intervalMs = (_cfg->timers[i].hours   * 3600UL +
                           _cfg->timers[i].minutes * 60UL   +
                           _cfg->timers[i].seconds) * 1000UL;
    
    // Проверка: прошёл ли интервал?
    if (now_ms - _timerLastTick[i] >= intervalMs) {
        _timerLastTick[i] = now_ms;
        // Выполнить действие таймера
        executeTimerAction(i);
    }
}
```

## Доступные действия (Action)

| ID  | Название       | Описание                                    |
|-----|----------------|---------------------------------------------|
| 0   | OFF            | Выключить светильник                        |
| 1   | GROW           | Режим роста (R70% FR50% B50% W30%)         |
| 2   | FULL           | Полная яркость (все каналы 100%)            |
| 3   | RED            | Только красный 100%                         |
| 4   | BLUE           | Только синий 100%                          |
| 5   | PULSE_GROW     | Импульс GROW (включить на duration_ms)    |
| 6   | PULSE_FULL     | Импульс FULL (включить на duration_ms)    |
| 7   | CUSTOM         | Свои значения каналов (R/FR/B/W)            |
| 8   | PULSE_CUSTOM   | Импульс с своими значениями                 |

## Параметры таймера

Каждый таймер имеет следующие настраиваемые параметры:

| Параметр       | Описание                                               |
|----------------|--------------------------------------------------------|
| **enabled**    | Включён/выключен                                      |
| **label**      | Имя/описание таймера (до 15 символов)                |
| **hours**      | Интервал: часы (0-23)                                 |
| **minutes**    | Интервал: минуты (0-59)                               |
| **seconds**    | Интервал: секунды (0-59)                              |
| **duration_ms**| Длительность импульса в миллисекундах (для PULSE-режимов) |
| **action**     | Действие при срабатывании (см. таблицу выше)         |
| **red**        | Красный канал для CUSTOM (0-200)                     |
| **far_red**    | Дальний красный для CUSTOM (0-200)                   |
| **blue**       | Синий канал для CUSTOM (0-200)                       |
| **white**      | Белый канал для CUSTOM (0-200)                       |
| **run_hours**  | Время автовыключения: часы (0-23)                     |
| **run_minutes**| Время автовыключения: минуты (0-59)                   |
| **run_seconds**| Время автовыключения: секунды (0-59)                  |

## Примеры использования

### Пример 1: Вечернее включение на 2.5 часа
- Интервал: 0 часов, 15 минут, 0 секунд (каждые 15 минут)
- Действие: FULL (полная яркость)
- Время работы: 2 часа, 30 минут, 0 секунд (автовыключение)

Логика:
```
14:00:00 → Таймер сработал → Светильник ВКЛ (FULL)
14:15:00 → Таймер сработал → Светильник ВКЛ (FULL)
...
16:30:00 → Автовыключение (run_time истёк) → Светильник ВЫКЛ
```

### Пример 2: Утренний импульс красного
- Интервал: 0 часов, 30 минут, 0 секунд (каждые 30 минут)
- Действие: PULSE_RED
- Длительность импульса: 500 мс

Логика:
```
08:00:00 → RED ВКЛ → через 500мс → RED ВЫКЛ
08:30:00 → RED ВКЛ → через 500мс → RED ВЫКЛ
...
```

### Пример 3: Свой режим с автовыключением
- Интервал: 1 час, 0 минут, 0 секунд (каждый час)
- Действие: CUSTOM
- Каналы: R=100, FR=50, B=0, W=80
- Время работы: 0 часов, 45 минут, 0 секунд

## Важные особенности

### Независимость от NTP/интернета
Интервальные таймеры используют `millis()` — внутренний счётчик времени ESP32, который:
- Работает сразу после включения
- Не требует интернета
- Не требует NTP синхронизации
- Сбрасывается при перезагрузке ESP32

### Автовыключение (Run Duration)
Если задано время работы (run_hours/run_minutes/run_seconds):
- Светильник автоматически выключится после указанного времени
- Это полезно для автоматического отключения после рабочего цикла

### Импульсные режимы (PULSE)
Режимы с префиксом `PULSE_`:
- Включают светильник на `duration_ms` миллисекунд
- Автоматически выключают после истечения времени
- Идеально для мигающих сигналов или коротких вспышек

## Ограничения

- **Перезагрузка**: При перезагрузке ESP32 таймеры сбрасываются (начинают заново)
- **Дрейф**: Точность интервалов зависит от стабильности кварца ESP32 (~±0.5% за час)
- **Максимум 8 таймеров**: Доступно 8 независимых слотов

## Сравнение систем таймеров

| Параметр          | Сценарии (по времени)     | Интервальные таймеры       |
|-------------------|--------------------------|---------------------------|
| **Триггер**       | Конкретное время суток   | Интервал (h:m:s)          |
| **Источник времени** | NTP / Backup Timer    | millis()                  |
| **Требует интернет** | Да (для точного NTP) | Нет                       |
| **Работает после реboot** | Нет (сбрасывается) | Нет (сбрасывается)        |
| **Пример**        | "В 08:00 включить"       | "Каждые 15 минут включить" |
| **Количество**    | 8 слотов                 | 8 слотов                  |
